What's New in Version 20 of Solid Edge Embedded Client

Version 20 implements Teamcenter's Service-Oriented Architecture (SOA) and File Management System (FMS) and demonstrates the ever increasing value of the Solid Edge product. This implementation can be used  with Teamcenter's 4-Tier deployment, where SOA is a component of this solution, or it can be configured to be used with a Teamcenter 2-Tier deployment.

In a 4-Tier deployment, only the client application executes on the users' workstation; the Enterprise tier, database, volume, and other resource tier components are executed on the server giving improved performance for customers working in a high-latency WAN environment. If a high-latency WAN is not a factor, but rather a larger number of users, you should consider a 2-tier configuration.  By default, SEEC V20 is a 4-tier configuration. For more details on 2-tier, refer to the Configuring Solid Edge Embedded Client for 2-Tier Deployment section of this readme file.)

In addition to SOA/FMS, many other new features have been added to this release, all of which continue the focus on implementing state-of-the-art technology with outstanding usability. For a complete list of V20 enhancements, start Solid Edge and choose "What's New" from the Help menu or choose the "What's New" topic under the "Solid Edge V20 Release Notes" available on the Support Web Page at http://support.ugs.com/docs/solid-edge.html.

Installing Solid Edge Embedded Client Administrator

The Solid Edge Embedded Client Administrator installation is required for Teamcenter Express (TcX) and Teamcenter Engineering (TcE).  Solid Edge Embedded Client Administrator is installed on the Teamcenter server only.

The Solid Edge Embedded Client Administrator installation delivers prerequisite files to the Teamcenter application directory and to the Teamcenter data directory, and can optionally configure the database and load Solid Edge templates. The user installing SEEC Administrator must have write access to both the Teamcenter application directory (TC_ROOT) and the Teamcenter Data directory (TC_DATA). For this reason, it is recommended that this installation be performed on the server machine where Teamcenter is installed.

Teamcenter Deployment Troubleshooting

Sizing 4-Tier

While using SEEC as a 4-Tier application, you must configure the Teamcenter middle tier with enough Teamcenter Business Logic Server processes to handle the incoming requests. You cannot arbitrarily set these numbers, as this will adversely impact reliability and performance. If the specified numbers are set too low, you will see reliability issues with Solid Edge operations that interact with Teamcenter, such File-Open, File-Save, and so forth. If the numbers are set too high, then performance is impacted. The following information is provided to help educate and guide you in configuring your server processes.  

For Teamcenter Engineering, the Server Manager configuration is managed in %TC_ROOT%\pool_manager\serverPool*.properties file. The file entries in which you are most interested are PROCESS_MAX, PROCESS_WARM, and PROCESS_TARGET.

  • PROCESS_MAX  Specifies the maximum number of Teamcenter Business Logic Server processes allowed to run in the pool.  The total number of servers will never exceed this number.

For example: PROCESS_MAX=30

 

In this example, the total number of servers will never exceed 30. 

PROCESS_MAX should always be larger than PROCESS_TARGET and PROCESS_WARM.

  • PROCESS_WARM  Specifies the minimum number of Business Logic Server processes in the pool that are started, but not logged into.  The server manager always tries to maintain this minimum number of servers ready (warm).  The Server Manager may time out servers in use, in order to ensure that there are at least the required number of minimum warm servers at all times.

For example: PROCESS_WARM=1

 

This ensures that one (1) server is always ready (warm), in addition to those being used.

  • PROCESS_TARGET Specifies the target number of server processes to be available in this pool during specified times. Specify these values as time and integer pairs separated by commas.

For example: PROCESS_TARGET=0700 3, 1700 2

This value sets the target number of server processes as 3 between 7:00 a.m. and 5:00 p.m. and as 2 between 5:00 p.m. and 7:00 a.m.

If the number of server processes is below the specified target, then warm servers are added to reach this number. In this case, the number of warm servers exceeds the minimum, so none are added.  If the number of server processes exceeds the specified target, only the minimum number of warm servers is maintained and servers are terminated as they time out.

 

For example, if you expect 10 concurrent users, you would want to set

PROCESS_WARM=1

PROCESS_TARGET=0700 11

This would keep 10 servers running and 1 ready (warm) at all times.

 

Depending on the server load and activity, these values should be modified. However, this must be considered with respect to the amount of physical memory and the other work this server is expected to perform. You should also consider the following:

  • Number of concurrent Teamcenter logins from all 4-tier enabled clients (SEEC, RAC, WebClient, and so forth)

  • Are you in the data migration phase of the project (AddtoTeamcenter) - where possibly more servers may be needed.

For Teamcenter Express, the Server Manager configuration can be managed via a web page. For example, http://myteamcenter.ugs.com:8080/tc/admin, where you would substitute your server in the URL. After a successful login, the Global Configuration page is displayed. Select the Server Manager pool in the Server Manager Instances section. (State should be running.) The Pool Configuration is displayed and can be updated using Edit. Here, Process Target is the parameter of interest.

Consult the Teamcenter documentation for additional details and information on other parameters.

Working with Teamcenter Managed NX Content

With the adoption of Teamcenter SOA, SEEC V20 has had to address how it works with Teamcenter managed NX content. This section highlights the SEEC V20 functionality.

·         SEEC V20 does not require an installation of NX/NXM.

·         SEEC displays content save to a Dataset=UGMASTER.

·         SEEC expects a data model where the UGMASTER Dataset is connected to an Item Revision with Relation=Specification.

·         SEEC V20 has been tested with NX4 and NX5 version documents.

·         No additional configuration is required to work with NX managed content.

·         SEEC does not update the NX content.

·         SEEC does not pull Properties from the NX document.

Note: On the File:Open dialog, Preview displays the Quick Access Binary file under the UGMASTER Dataset. This was created by default in NX4 and appears to be missing in NX5.

Opening an NX Part

·         File:Open is a non-associative workflow.

·         An SEEC Save does not populate the New Document dialog with any NX Property data. The Designer can save this Solid Edge document to the same Item Revision as the NX document by typing in the Item ID and Revision or assign a new Item ID and Revision.

·         When a Solid Edge Dataset and UGMASTER Dataset exist in the same Item Revision, SEEC will download both Datasets.

Insert:Part Copy, select an NX Part

·         Insert:Part Copy is an associative workflow. (Creating associativity is an option on the Part Copy command.)

·         An SEEC Save does not populate the New Document dialog with any NX Property data. The Designer can save this Solid Edge document to the same Item Revision as the NX document by typing in the Item ID and Revision or assign a new Item ID and Revision. If a new Item ID and Revision is assigned, a Relation is created between the two to capture this design intent.

·         When a Solid Edge Dataset and UGMASTER Dataset exist in the same Item Revision, SEEC will download both Datasets.

Opening an NX Assembly

·         When the selected UGMASTER is connected to an Item Revision that has a View Type (as defined by the NX Preference), SEEC will expand that view per the Revision Rule and give Solid Edge only those Item Revisions that contain UGMASTER Datasets.

·         Solid Edge will present the managed Templates. The expected selection should be a Solid Edge Assembly template. At this point, Solid Edge will execute the same sequence of steps as the unmanaged workflow.

·         SEEC will present the New Documents dialog for each subassembly it processes. Each Solid Edge Assembly MUST BE saved to a new Item ID and Revision! It must not be saved to the same Item Revision as the input NX Dataset.

·         When the Solid Edge Assembly is closed, if the user selects No to the Save, the default behavior is to remove the Solid Edge files. This is consistent with the unmanaged behavior. The only impact to this is, any Item IDs assigned to the sub-assemblies are not present in Teamcenter.

·         When the Solid Edge Assembly is Closed and Saved, the New Document dialog is presented, requiring Item ID and Revision information for the new Solid Edge content. By default, the NX content was Part Copied into the Solid Edge file to maintain associativity. These Solid Edge Parts could be saved to the save Item Revisions as the NX content, or assigned new Item ID and Revision.

Note: SEEC does not have a mechanism to take structure updates from the managed NX Assembly.

Attribute Mapping:

Attribute mapping defines what document properties exchanged between Solid Edge and Teamcenter. In Teamcenter, the key attributes normally used to track part numbers and revisions are Item ID and Revision ID. Each of these Teamcenter attributes are limited to 32 characters. In addition to these two properties, Teamcenter also requires an Item Type and Item Name. SEEC additionally maps Dataset Description, providing another property to capture additional detail about the specific Solid Edge document. Your unmanaged data's properties should be reviewed and prepared for importing before the content is uploaded to Teamcenter.

Silent Installation of Solid Edge Embedded Client

The following is a complete command-line argument set for silently installing Solid Edge Embedded Client (Client installation).

All arguments should be encased in double quotes if there are spaces in the paths.

C:\>msiexec /i "<DVD>\Embedded Client\Solid Edge Embedded Client V20.msi"
    INSTALLDIR="C:\Program Files\Solid Edge V20"
    TCTYPE="TCX"
    TCVERSION=1
    TCDBURL="http://myteamcenter.ugs.com:8080/tc"
    TCDBDESC="Production"
    SPSACFILE="\\myteamcenter\UGS\Solid Edge Standard Parts\SMAPINIFile.sac"
    TCDBDEFAULT=1
    /qn+ /l*v c:\SEECv20_Client.log

where

<DVD> is your DVD drive letter (D:), a mapped network drive letter and path or a UNC path to the installation files.
<Computer Name> is the name of the Computer sharing Teamcenter Data (formerly, IMAN_DATA)

SEEC Client public properties are:

·         INSTALLDIR - Installation location for Embedded Client and in quotes (this should be the Solid Edge V20 installation folder)

·         TCTYPE - TCX or TCE and in quotes

·         TCVERSION - Teamcenter version; 1 for TcE2007/TcXv2.2

·         TCDBURL - Teamcenter Database URL and in quotes, Required

·         TCDBDESC - Teamcenter Database Description and in quotes, Required

·         SPSACFILE - If Solid Edge Standard Parts were configured for use with Teamcenter, enter the SAC file configured for this Teamcenter Database, Optional

·         TCDBDEFAULT - Default Teamcenter Database; 0 for FALSE or 1 for TRUE, Optional

For additional help on msiexec switches/options, from a DOS Prompt, C:> msiexec

Silent Installation of Solid Edge Embedded Client Administrator

The following is a complete command-line argument set for silently installing Solid Edge Embedded Client (Administrator installation). All arguments should be encased in double quotes if there are spaces in the paths.

C:\>msiexec /i "<DVD>\SEEC Administrator\SEEC Administrator V20.msi"
    INSTALLDIR="C:\UGS\2005SR1"
    TCROOTDIR="C:\UGS\2005SR1"
    TCDATADIR="C:\UGS\tcdata"
    TCVERSION=1
    TEMPLATEFOLDER="c:\progra~1\solide~1\Program\Templates"
    DBUSERNAME="infodba"
    DBPASSWORD="infodba"
    ADDLOCAL=ALL /qn+ /l*v c:\SEEC_Administrator.log

where
<DVD> is your DVD drive letter (D:), a mapped network drive letter and path or a UNC path to the installation files.
 

SEEC Administrator public properties are:

·         INSTALLDIR - Installation location for SEEC Administrator and in quotes (recommend TC_ROOT)

·         TCROOTDIR - Teamcenter Application folder (TC_ROOT) in 8.3 file format and in quotes

·         TCDATADIR - Teamcenter Data folder (TC_DATA) in 8.3 file format and in quotes

·         TCVERSION - Teamcenter version; 1 for TcE2007/TcXv2.2 or 2 for Teamcenter 2007                                                                                                    

·         TEMPLATEFOLDER - Provide Solid Edge Template folder in 8.3 format in quotes

·         DBUSERNAME - Username for Teamcenter, default=infodba

·         DBPASSWORD - Password for Teamcenter database, default=infodba

·         ADDLOCAL - Valid inputs include: ALL or ProgramFiles or ConfigureDB or ImportTemplates (example: ADDLOCAL=ImportTemplates)  

For additional help on msiexec switches/options, from a DOS Prompt, type C:> msiexec /h
